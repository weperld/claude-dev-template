description: 간편 작업 - 파이프라인 생략, 최소 검증만 수행
---

다음 작업을 개발 파이프라인 없이 수행해주세요:

[작업 내용]: $ARGUMENTS

**모드 설명**:
개발 파이프라인을 생략합니다.
WorkID, WIP 추적, Cross-Stage Review, Gate 검증을 수행하지 않습니다.

**최소 필수 사항** (생략 불가):
1. **작업 전 코드 읽기**: 수정 대상 파일을 반드시 읽고 이해한 후 작업
2. **절대 규칙 준수**: AGENTS.md의 Hard Blocks는 항상 적용
3. **빌드 검증**: 코드 변경 시 빌드 에러 0건 확인
4. **검증 항목 확인**: `.guides/VERIFICATION_ITEMS.md`의 해당 체크리스트 확인
5. **작업 완료 보고**: 변경 파일 목록, 변경 내용 요약, 검증 결과를 사용자에게 보고

**적합한 작업 유형**:
- 프로젝트 구조 변경 (패키지화, 폴더 이동 등)
- 설정 파일 수정
- 단순 리팩터링 (이름 변경, 파일 정리 등)
- 문서 작성/수정
- 빌드/배포 관련 작업

**부적합한 작업 유형** (파이프라인 사용 권장):
- 새로운 기능 추가 → `/project:신규` 사용
- 버그 수정 → `/project:수정` 사용
- 복잡한 로직 변경

**문제 발생 시**:
- 작업 중 발견된 문제가 재발 가능성이 있으면 `.guides/VERIFICATION_ITEMS.md`에 검증 항목 추가

**예시**:

```bash
/project:간편 프로젝트 폴더 구조 재정리
→ 파이프라인 생략, 직접 작업 수행
→ VERIFICATION_ITEMS.md 체크리스트 확인
→ 빌드 검증 수행
→ 변경 내용 보고
```
