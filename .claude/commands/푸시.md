---
description: 푸시 - 원격 저장소로 푸시
---

다음 메시지로 푸시를 진행해주세요:

[푸시 메시지]: $ARGUMENTS

**작업 절차**:
1. 상태 확인 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
2. 로컬 커밋이 있는지 확인
3. 메시지 처리:
     - 메시지가 있는 경우: 지정된 메시지로 푸시
     - 메시지가 없는 경우: git log로 최근 커밋 분석하여 메시지 자동 생성
4. 원격 저장소로 푸시
5. 푸시 성공 확인

**메시지 자동 생성 규칙** (메시지가 없는 경우):
1. git log로 최근 커밋 확인 (최근 1-3개)
2. 커밋 메시지 분석 (태그, 요약, 유형) → COMMIT_RULES.md의 커밋 태그 섹션 참조
3. 푸시 메시지 생성:
     - 단일 커밋: 해당 커밋 메시지의 요약 부분 사용
     - 여러 커밋: 주요 변경 사항 종합하여 요약
     - 형식: 간결한 요약 (한글, 50자 이내 권장)

**푸시 메시지 형식**: 커밋과 유사한 형식 → COMMIT_RULES.md의 기본 형식 섹션 참조

**요구사항**:
- 기본 요구사항 → AGENTS.md의 절대 규칙 섹션 및 COMMIT_RULES.md의 커밋 및 푸시 안전 규칙 섹션 참조

**예시**:

사용자 지정 메시지:
```
/project:푸시 CSV 기능 추가 및 버그 수정
/project:푸시 [refactor] 아키텍처 개선
```

메시지 없는 경우 (자동 생성):
```
/project:푸시
→ 분석 결과: 최근 1개 커밋 "[feat] CSV 데이터 추출 기능 추가"
→ 생성 메시지: CSV 데이터 추출 기능 추가
```
