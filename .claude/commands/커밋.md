---
description: 커밋 - 로컬 변경 사항 커밋
---

다음 메시지로 커밋을 진행해주세요:

[커밋 메시지]: $ARGUMENTS

**작업 절차**:
1. 변경 사항 확인 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
2. 변경 내용 검토 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
3. 변경 파일 스테이징 (add) → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
4. 메시지 처리:
     - 메시지가 있는 경우: 지정된 메시지로 커밋
     - 메시지가 없는 경우: 변경 사항을 분석하여 [태그] 요약 형식으로 메시지 자동 생성
5. 커밋 확인 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조

**메시지 자동 생성 규칙** (메시지가 없는 경우):
1. 변경 파일 타입 분석 → COMMIT_RULES.md의 커밋 태그 섹션 참조
2. 변경 내용 요약 및 형식 → COMMIT_RULES.md의 작성 규칙 섹션 참조
3. WorkID 포함 (해당 WIP가 있는 경우) → COMMIT_RULES.md의 WorkID 참조 섹션 참조

**커밋 메시지 기본 형식**: COMMIT_RULES.md의 기본 형식 섹션 참조

**요구사항**:
- 기본 요구사항 → AGENTS.md의 절대 규칙 섹션 및 COMMIT_RULES.md의 커밋 및 푸시 안전 규칙 섹션 참조

**예시**:

사용자 지정 메시지:
```
/project:커밋 [feat] CSV 데이터 추출 기능 추가
/project:커밋 [fix] ExportService null 참조 수정
```

메시지 없는 경우 (자동 생성):
```
/project:커밋
→ 분석 결과: ExportService.cs 추가, ExportViewModel.cs 수정 (새로운 기능)
→ 생성 메시지: [feat] CSV 데이터 추출 기능 추가
```
