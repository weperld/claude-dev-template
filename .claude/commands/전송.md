description: 전송 - 스테이징, 커밋, 푸시 한번에 진행
---

로컬 변경 사항을 스테이징, 커밋, 푸시까지 한번에 진행해주세요.

**작업 절차**:
1. 변경 사항 확인 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
2. 변경 내용 검토 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
3. 변경 사항 분류:
     - 관련 내용별로 그룹화 (예: CSV 기능 관련 파일, 버그 수정 관련 파일)
     - 파일 내용, 경로, 변경 타입 분석
4. 각 그룹별로 스테이징:
     - 관련 파일들을 함께 add → COMMIT_RULES.md의 기본 작업 절차 섹션 참조
     - 그룹별로 별도 커밋 생성
5. 커밋 메시지 자동 생성 → COMMIT_RULES.md의 커밋 메시지 관련 섹션 참조
6. 모든 커밋 완료 후 원격 저장소로 푸시
7. 최종 상태 확인 → COMMIT_RULES.md의 기본 작업 절차 섹션 참조

**변경 사항 분류 및 그룹핑 규칙**: COMMIT_RULES.md의 커밋 분류 및 그룹핑 규칙 섹션 참조

**태그 결정 규칙**: COMMIT_RULES.md의 커밋 태그 섹션 참조

**커밋 메시지 작성 규칙**: COMMIT_RULES.md의 커밋 메시지 관련 섹션 참조

**요구사항**:
- 기본 요구사항 → AGENTS.md의 절대 규칙 섹션 및 COMMIT_RULES.md의 커밋 및 푸시 안전 규칙 섹션 참조

**예시**:

```bash
# 단일 그룹 변경
/project:전송
→ 분석 결과: ExportService.cs 추가, ExportViewModel.cs 수정 (CSV 기능)
→ 커밋 1: [feat] CSV 데이터 추출 기능 추가
→ 푸시 완료

# 여러 그룹 변경
/project:전송
→ 분석 결과:
  1. ExportService.cs, ExportViewModel.cs (CSV 기능 추가)
  2. ExportServiceTests.cs (테스트 추가)
  3. README.md (문서 업데이트)
→ 커밋 1: [feat] CSV 데이터 추출 기능 추가
→ 커밋 2: [test] CSV 추출 기능 테스트 추가
→ 커밋 3: [docs] CSV 기능 관련 문서 업데이트
→ 푸시 완료
```
